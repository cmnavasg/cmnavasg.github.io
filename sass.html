Sass corresponde a las siglas de Syntactically Awesome StyleSheets y es una extensión de CSS. Le permite utilizar variables, reglas anidadas, importaciones y mucho más. Es totalmente compatible con CSS y ayuda a que las hojas de estilos grandes puedan estar más organizadas y a las pequeñas les otorga rapidez.

Existen dos sintaxis disponibles para Sass. El primero es SCSS (Sassy CSS) y es una extensión de la sintaxis de CSS. Scss entiende la sintaxis específica de CSS y ha mejorado con las características propias de SASS. Los archivos que contienen esta sintaxis utilizan extensión .scss

La segunda es más antigua y es conocida como Sass y es más concisa a la hora de escribir CSS. Utilizan las nuevas líneas en lugar de punto y coma para separar las propiedades, así como la sangría en lugar de los corchetes para el anidamiento de selectores. Los archivos que utilizan esta sintaxis tienen .sass por extensión.

Un archivo preprocesado sass es capaz de entender lenguaje css, scss y sass propiamente. Una vez se tenga el archivo preprocesado sass, será necesario compilarlo y pasará a ser un archivo css que se puede añadir al proyecto web para que los navegadores sean capaces de entenderlo.

Si se considera que hay determinados elementos (fuentes, colores...) que se van a utilizar a lo largo de toda la aplicación web, se pueden utilizar variables. Para identificar las variables Sass utiliza el carácter $ precediendo el nombre de variable.

Sass además permite anidar los selectores de CSS de manera que siga la misma jerarquía visual que HTML. Por otro lado, el exceso de anidamiento es difícil de mantener por lo que se considera que es una mala práctica.

Una de las cosas más atractivas de Sass es la capacidad de herencia. De manera que, permite compartir un conjunto de propiedades de un selector a otro evitando escribir propiedades repetidas y lo que supone tener menos líneas de código y más legibles.
